# grpc-store-stub

## 目的
今後、gRPCからの大量のレスポンスデータを格納する機能を実装する。
この際、データ量などを模擬できるスタブを実装する。

## スタブの内部実装
スタブの内部処理は、等速直線運動をする多数オブジェクトのシミュレーションを行います。
grpcのリクエストで、オブジェクトの初期位置とエリアを指定が行われるため、
すべてのオブジェクトは等速直線を行いエリアに衝突すると反射する。
シミュレーション内の演算周期は、1秒周期とする。ただし、実時間である必要はない。

## gRPCでのインタフェースについて
grpcでは、sample_request.jsonと同等のgRPCのリクエストを受け取り、sample_reeponse.jsonと同等のデータを返す動作を想定している。
このJSON相当のprotoファイルの作成が必要である。

### 送信方式について

gRPCの実装は、server streaming とする。
リクエストを受け付け、一定周期で、レスポンスを返すこととする。
最後のレスポンスでは、isFinalがtrueになる。


### リクエスト構造について
リクエストではシミュレーションの条件が入ってる。

#### objects
objectsでは、オブジェクトの初期位置と、速度・方向が記載されている。
このデータの数を変更することも実装確認の１つである。
なお、位置は、3次元であるが、これはデータ量の調整のためであり、演算上は2次元で実施してよい。

#### area
areaは、検証そのものには関係ないが、長時間動作をさせる際に、数値が範囲外に落ちないように一定区画を動き続ける制約として設けている。

#### range
rangeは、シミュレーション時間の開始秒から終了秒である。
この時間によって、データの出力時間を制御する。

#### interval
intervalは、シミュレーション上の計算間隔（秒）である。
これは実時間ではなくシミュレーション上の時間であるため、実時間に関係なく進む

#### bulkSize
これはserver streamingで返すデータ数である。
interalが1秒で、bulkSizeが、100の場合、計算が100回、シミュレーション時間上で100秒に一回レスポンスを返す形になります。

### wait 
waitは、送信の時間間隔（秒）である。
これは、計算を簡素にしたため、送信周期が早くなりすぎないように、sleepを入れるための値である。
演算に一定の処理時間が係る場合は、0のまま運用する。


### events構造について
なお、eventsは、受信型で多言語メッセージを出力するために使われるため、メッセージ種別とそのメッセージに対するプレースホルダ変数を与える形としている。
受信側では、以下のような言語ファイルを持ち、
```
{
  "m0001": "Hello, {name}. I am {age} years old.My height is {height} cm."
}
```
送信側で、を送信すると、
```
{
    "type": "m0001",
    "args": {
        "age": 17,
        "height": 163.5,
        "name": "Jane"
    }
}
```
以下のようなメッセージとなる想定です。
```
Hello, Jane I am 17 years old.My height is 163.5 cm.
```

protoファイルを作成する際に、この点に留意してください。
ただし、現時点では自動生成されたコードの使い方を検証するのみのため、本stubからのデータ転送は不要とする。
